# 第一部分：算法设计

## 1. 算法名称

残差驱动自适应加权 A* 路径规划算法（Residual-Driven Adaptive Weighted A*）

## 2. 解决的问题

标准 A* 在 8 邻域栅格地图中存在三个问题：
1. 欧氏启发函数低估代价 → 扩展大量无效节点 → 搜索慢
2. 加权 A* 的权重 α 是固定值或地图级值 → 高障碍地图 α=1.0 完全无效
3. 输出路径沿栅格走 → 锯齿严重 → 机器人难以直接执行

## 3. 核心公式

```
f(n) = g(n) + α(n) × h_oct(n)

α(n) = 1 + β × (1 − ρ_local(n))
```

| 符号 | 含义 | 取值 |
|------|------|------|
| g(n) | 起点到 n 的实际代价 | 逐步累加（直走+1，对角+√2） |
| h_oct(n) | Octile距离 = max(dx,dy) + (√2−1)×min(dx,dy) | 8邻域下最紧的可采纳下界 |
| ρ_local(n) | 节点 n 周围 11×11 窗口的障碍率 | [0, 1]，积分图 O(1) 查询 |
| β | 全局修正系数 | 建议 0.3，范围 [0, 0.5] |
| α(n) | 节点级自适应权重 | 范围 [1.0, 1+β] |

**行为：**
- ρ_local = 0（纯空地）→ α = 1 + β = 1.3 → 最大加速
- ρ_local = 0.5（中等障碍）→ α = 1 + 0.5β = 1.15 → 中等加速  
- ρ_local = 1（纯障碍）→ α = 1.0 → 退化为标准 A*

## 4. 三个创新点

### 创新点 1：Octile 启发函数替代欧氏距离

**问题：** 欧氏距离在 8 邻域下低估代价。机器人只能走 8 个方向（步长 1 或 √2），不能沿任意方向直飞。

**方案：** h_oct(n) = max(dx,dy) + (√2−1)×min(dx,dy)，是 8 邻域下最紧的可采纳下界。

**原理：** 先沿对角走 min(dx,dy) 步（每步 √2），再沿直线走 |dx−dy| 步（每步 1）。这恰好是无障碍假设下的最短路径代价。h_oct ≥ h_euc 且 h_oct ≤ h*（真实代价），启发更紧 → A* 优先级判断更准 → 无效扩展更少。

**效果：** 扩展节点减少约 25-42%（相对欧氏基线）。

### 创新点 2：残差驱动的节点级自适应权重（核心创新）

**问题：** 
- 固定 α（如 α=1.2）：不区分地图和节点
- 地图级 α（如 clip(|ln(ρ)|, 1.0, 1.8)）：15 张地图中 9 张 α=1.0 完全无效

**方案：** α(n) = 1 + β × (1 − ρ_local(n))

**原理（残差学习思想推导）：**

```
真实代价分解: h*(n) = h_oct(n) + Δh(n)
  
  h_oct(n) = 无障碍假设下的精确代价（已知基准）
  Δh(n) = 障碍物造成的额外绕行代价（残差，≥ 0）

关键洞察: 
  ρ_local 高 → 障碍多 → 绕行多 → 残差 Δh 大 → h_oct 严重低估 → 不敢放大α
  ρ_local 低 → 障碍少 → 绕行少 → 残差 Δh 小 → h_oct 接近 h* → 可以安全放大α

因此: α 应与 (1 − ρ_local) 正相关
结果: α(n) = 1 + β × (1 − ρ_local(n))
```

**与现有方法的对比：**

| 方法 | α 来源 | 高障碍地图效果 | 文献 |
|------|--------|-------------|------|
| 固定 α | 人为常数 | 全图一个值 | Pohl 1970 |
| 地图级 α | 全局障碍率 | 9/15 地图 α=1.0 无效 | 本文旧版 |
| **节点级 α（本文）** | **局部障碍率** | **走廊空地仍 α>1** | **无先例** |

**效果（β=0.3，最长路径任务）：**

| 地图 | 障碍率 | 旧方案 vs Octile | **本文方案 vs Octile** |
|------|--------|-----------------|---------------------|
| arena.map | 0.145 | -51% | **-50.0%** |
| brc000d | 0.568 | **0%（α=1.0失效）** | **-38.7%** |
| battleground | 0.648 | **0%（α=1.0失效）** | **-84.4%** |

### 创新点 3：两阶段路径后处理

**问题：** A* 输出路径沿栅格走，锯齿严重。

**方案：**
- 阶段 1（视线简化）：从锚点出发，Bresenham 直线检测到后续节点是否有无障碍视线。有视线则跳过中间点。
- 阶段 2（拐角插值平滑）：对剩余拐角做 p' = (p_{i-1} + 2p_i + p_{i+1})/4 加权平均。每次检测碰撞，穿墙则保留原点。

**效果：** 转弯次数减少 95-99%，路径长度缩短 4-5%。

## 5. 三个创新点的协作

```
输入: 栅格地图 + 起终点
  │
  ▼
[创新点1] Octile 启发函数 → 收紧启发估计
  │
  ▼
[创新点2] 残差驱动自适应权重 → 节点级搜索加速
  │         空旷区域 α=1.3 → 快速推进
  │         密集区域 α≈1.0 → 保守搜索
  ▼
A* 搜索输出原始路径（锯齿状）
  │
  ▼
[创新点3] 两阶段平滑 → 删冗余点 + 平滑拐角 + 碰撞检测
  │
  ▼
输出: 平滑路径（转弯-96%，路径-5%）
```

# 第二部分：几何合法性保证

| 规则 | 实现方式 | 依据 |
|------|---------|------|
| 路径不穿墙 | 每个节点 grid[x,y] ≠ 1 | 基本要求 |
| 对角不切角 | neighbors8: grid[x+dx,y]==1 OR grid[x,y+dy]==1 则禁止对角 | MovingAI: "cannot cut corners through walls" |
| 视线简化合法 | Bresenham line_of_sight 逐段检测 | — |
| 平滑后合法 | 新点碰撞检测 + 前后连线 line_of_sight 双检 | — |
| BFS一致性 | _is_reachable 使用相同的 anti-tunneling 规则 | 与 neighbors8 一致 |

**已验证：** Octile A* 路径长度与 MovingAI 官方最优值精确匹配（arena2.map 30/30），与 NetworkX Dijkstra 三方一致。

# 第三部分：实验设计

## 对比组（7组）

| # | 算法 | 启发函数 | 权重 | 平滑 | 作用 |
|---|------|---------|------|------|------|
| 1 | Dijkstra | 无(h=0) | — | 无 | 无启发基线 |
| 2 | A*(欧氏) | 欧氏 | α=1.0 | 无 | **主基线** |
| 3 | A*(Octile) | Octile | α=1.0 | 无 | 隔离创新点1 |
| 4 | 加权A* | Octile | α=1.2(固定) | 无 | 固定权重参照 |
| 5 | **改进A*(本文)** | **Octile** | **α(n)=1+β(1-ρ)** | **两阶段** | **完整方案** |
| 6 | 消融:无自适应 | Octile | α=1.0 | 两阶段 | 隔离创新点2 |
| 7 | 消融:无平滑 | Octile | α(n)=1+β(1-ρ) | 无 | 隔离创新点3 |

## 数据口径

- **地图：** MovingAI Lab 15 张真实地图
  - DAO（Dragon Age Origins）× 5：arena, arena2, brc000d, brc100d, brc101d
  - Street（城市街道）× 5：Berlin_0_256, Berlin_0_512, Berlin_0_1024, Berlin_1_256, Berlin_1_1024
  - WC3（Warcraft III）× 5：battleground, blastedlands, bloodvenomfalls, bootybay, darkforest
- **任务：** 每张地图的官方 .scen 文件，取 30 个任务（strict scen，固定起终点）
- **对角规则：** OnlyWhenNoObstacles（AND 逻辑），与 MovingAI 口径一致
- **重复：** 确定性算法无随机性，单次运行即可

## 统计方法

| 方法 | 用途 | 工具 |
|------|------|------|
| 均值 ± 标准差 | 描述统计 | numpy |
| 配对 Wilcoxon 符号秩检验（双侧） | 推断统计（15 图配对） | scipy.stats.wilcoxon |
| BH-FDR 多重比较校正 | 控制假阳性 | statsmodels.multipletests |
| Cohen's d | 效应量 | 手动计算 |
| rank-biserial r | 效应量 | r = 1 - 2W/[n(n+1)/2] |

## 指标

- Runtime (ms)
- Path Length（欧氏距离累加）
- Turn Count（方向变化次数）
- Expanded Nodes（扩展节点数）
- Success Rate
- Collision-Free Rate（平滑后路径合法率）

# 第五部分：已知风险

| # | 风险 | 严重度 | 处理方式 |
|---|------|--------|---------|
| H1 | 基线A*用欧氏（PathFinding.js标准是Octile） | 中 | 论文说明："传统文献常用欧氏"，且实验含Octile A*对比组 |
| H2 | α(n)>1 时不保证严格最优（α-最优，路径≤(1+β)×最优） | 中 | 论文诚实说明 + 实验证明路径质量经平滑后反而更好 |
| H3 | β 需要标定 | 低 | 做参数敏感性实验（β=0.1~0.5），推荐 β=0.3 |
| H4 | 局部障碍率查询增加每节点开销 | 低 | 积分图预计算 O(HW) 一次，查询 O(1) |
